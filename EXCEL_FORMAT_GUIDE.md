# Excel文件格式详细说明

## 格式要求

### 列结构
Excel文件必须包含**前4列数据**（A、B、C、D列），程序会自动读取这4列并组合成文件夹名称。

### 数据格式

```
─────────────────┬────────┬──────────────┬───────────────┐
│ A列             │ B列    │ C列          │ D列           │
─────────────────┼────────┼──────────────┼───────────────┤
│ J76000016.3426  │ 雾卡其  │ 170/92A(M)   │ 6942674037033 │
│ J76000016.3427  │ 雾卡其  │ 175/96A(L)   │ 6942674037040 │
│ J76000016.3428  │ 雾卡其  │ 180/100A(XL) │ 6942674037057 │
│ J76000016.3429  │ 雾卡其  │ 185/104A(XXL)│ 6942674037064 │
│ J76000016.3430  │ 雾卡其  │ 190/108A(XXXL)│6942674037071 │
│ ...             │ ...    │ ...          │ ...           │
└─────────────────┴──────────────────────┴───────────────┘
```

## 命名规则

### 组合方式
程序会将4列数据用**下划线(_)** 连接起来，形成完整的文件夹名称。

**示例：**
```
A列: J76000016.3426
B列: 雾卡其
C列: 170/92A(M)
D列: 6942674037033

生成的文件夹名称:
J76000016.3426_雾卡其_170/92A(M)_6942674037033
```

### 特殊情况处理

1. **空单元格**
   - 如果某列为空，该列会被跳过
   - 示例：A列有值，B列为空，C列有值，D列有值
   - 结果：`A值_C值_D值`

2. **全部为空**
   - 如果某行4列都为空，该行会被完全跳过
   - 不会生成空的文件夹名称

3. **首尾空格**
   - 所有数据会自动去除首尾空格
   - 示例：`  雾卡其  ` → `雾卡其`

4. **特殊字符**
   - ️ 文件夹名称不能包含以下字符：`\ / : * ? " < > |`
   - 如果Excel中包含这些字符，重命名可能会失败

## Excel文件示例

### 示例1：标准格式

| A列 | B列 | C列 | D列 |
|-----|-----|-----|-----|
| J76000016.3426 | 雾卡其 | 170/92A(M) | 6942674037033 |
| J76000016.3427 | 雾卡其 | 175/96A(L) | 6942674037040 |
| J76000016.3428 | 雾卡其 | 180/100A(XL) | 6942674037057 |

**生成的文件夹名称：**
```
J76000016.3426_雾卡其_170/92A(M)_6942674037033
J76000016.3427_雾卡其_175/96A(L)_6942674037040
J76000016.3428_雾卡其_180/100A(XL)_6942674037057
```

### 示例2：包含空单元格

| A列 | B列 | C列 | D列 |
|-----|-----|-----|-----|
| J76000016.3426 | 雾卡其 | | 6942674037033 |
| J76000016.3427 | | 175/96A(L) | 6942674037040 |

**生成的文件夹名称：**
```
J76000016.3426_雾卡其_6942674037033          （B列为空，跳过）
J76000016.3427_175/96A(L)_6942674037040       （B列为空，跳过）
```

### 示例3：不同数据类型

Excel支持多种数据类型：

| A列（文本） | B列（文本） | C列（数字） | D列（文本） |
|------------|------------|------------|------------|
| Item001 | 产品A | 100 | ABC123 |
| Item002 | 产品B | 200 | DEF456 |

**生成的文件夹名称：**
```
Item001_产品A_100_ABC123
Item002_产品B_200_DEF456
```

## 文件格式

### 支持的格式
- ✅ `.xlsx` (Excel 2007及以上版本)
- ✅ `.xls` (Excel 97-2003版本)

### 推荐格式
推荐使用 `.xlsx` 格式，兼容性更好。

## 创建Excel文件

### 方法1：使用Microsoft Excel

1. 打开Microsoft Excel
2. 在A、B、C、D列输入数据
3. 点击"文件" → "另存为"
4. 选择保存位置
5. 文件类型选择"Excel工作簿 (*.xlsx)"
6. 点击"保存"

### 方法2：使用WPS表格

1. 打开WPS表格
2. 在A、B、C、D列输入数据
3. 点击"文件" → "另存为"
4. 选择保存位置
5. 文件类型选择"Excel文件 (*.xlsx)"
6. 点击"保存"

### 方法3：从CSV转换

如果有CSV文件：

1. 打开Excel
2. 点击"数据" → "从文本/CSV"
3. 选择CSV文件
4. 设置分隔符（通常是逗号）
5. 导入后检查数据是否正确分配到A、B、C、D列
6. 另存为.xlsx格式

## 注意事项

### ⚠️ 重要提醒

1. **列顺序固定**
   - 必须按照A、B、C、D的顺序
   - 程序只读取前4列
   - E列及以后的列会被忽略

2. **行数匹配**
   - Excel中的有效行数应 ≥ 需要重命名的文件夹数量
   - 如果Excel行数不足，会显示错误提示

3. **数据类型**
   - 文本类型：直接读取
   - 数字类型：自动转换为文本
   - 日期类型：转换为日期字符串
   - 公式：读取公式结果

4. **非法字符**
   - 文件夹名称不能包含：`\ / : * ? " < > |`
   - 如果Excel中包含这些字符，重命名会失败
   - 建议在Excel中使用替换功能预先处理

5. **文件名长度**
   - Windows系统文件名最大长度为260字符
   - 如果4列组合后超过限制，可能会失败
   - 建议控制每列数据的长度

### 常见问题

**Q1: Excel中有标题行怎么办？**
A: 程序会从第一行开始读取，如果有标题行，标题行也会被当作一个名称。建议删除标题行或将标题行放在第2行之后。

**Q2: 可以使用其他列吗？**
A: 目前版本只支持A、B、C、D前4列。如需使用其他列，需要修改源代码。

**Q3: 能否自定义连接符？**
A: 目前使用下划线(_)作为连接符。如需更改，需要修改源代码中的`loadExcelData()`方法。

**Q4: Excel中能有空行吗？**
A: 可以，程序会自动跳过全空的行。

**Q5: 数据中有空格怎么办？**
A: 程序会自动去除首尾空格，但中间的空格会保留。

## 验证Excel文件

### 检查清单

在重命名前，请检查：

- [ ] Excel文件可以正常打开
- [ ] A、B、C、D列包含有效数据
- [ ] 数据行数 ≥ 需要重命名的文件夹数量
- [ ] 数据中不包含非法字符（\ / : * ? " < > |）
- [ ] 组合后的名称长度不超过260字符
- [ ] 没有重要的空行或空白单元格影响数据

### 测试建议

1. 先创建一个包含2-3行数据的测试Excel文件
2. 准备2-3个测试文件夹
3. 执行程序并验证结果
4. 确认无误后再进行大批量操作
